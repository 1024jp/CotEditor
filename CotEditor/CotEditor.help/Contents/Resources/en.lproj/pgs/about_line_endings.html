Treating various types of line endings

CotEditor can treat various types of line endings. In general, CotEditor uses a single type of line endings for a document consistently.

Line ending characters in CotEditor

CotEditor treats the following characters as line ending delimiters.

LS Line Separator U+2028 A control character indicating the separator of lines in Unicode. Hidden by default.
PS Paragraph Separator U+2029 A control character indicating the separator of paragraphs in Unicode. Hidden by default.

The latter three types, NEL, LS, and PS, are hardly used in the real world as line ending delimiters for document files. Therefore, CotEditor hides them by default in the line ending choice.

In contrast, although VT (U+000B) and FF (U+000C) also visually breaks lines in CotEditor, they are not used as line endings characters.

Setting default Line endings

You can set the default line ending type for new documents in Preferences > Format > Default line endings.

Document line endings in an existing document

CotEditor detects the line ending used in a document automatically, and sets it as the line endings for the document.

If a document has inconsistent line endings, CotEditor automatically alerts and highlight them. For details, see Finding inconsistent line endings.

knowing document line endings

You can check the type of line endings used in the document either in the following area:

  • The label of the line endings button in the right side of the status bar
  • The Document Inspector by choosing View > Inspectors > Document.
  • The selection of the Line Endings submenu in the Format menu.
